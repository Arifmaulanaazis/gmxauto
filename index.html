gmxauto

A PyQt6-based GUI application for automating GROMACS 2025.1 molecular dynamics simulations on Windows, supporting both CUDA GPU and CPU builds. Designed to streamline running MD simulations prepared with CHARMM-GUI.

Features

  • Intuitive Windows GUI - User-friendly dark-themed interface built with PyQt6
  • GROMACS 2025.1 Support - Compatible with both CUDA GPU and CPU builds
  • Workflow Automation - Automates multi-step MD simulations from CHARMM-GUI inputs
  • Real-time Monitoring - Live progress tracking and output logging with colored status indicators
  • Checkpoint Handling - Automatically detects and resumes interrupted simulations
  • Performance Optimization - Smart GPU/CPU resource allocation and parameter tuning
  • Environment Management - Automatic configuration of GROMACS environment variables
  • Multi-threaded Processing - Background simulation execution with responsive UI

Architecture

gmxauto is built with a modular architecture that separates concerns for better maintainability:

Core Components

  • EnvironmentManager (environment_manager.py) - Configures system environment variables for optimal GROMACS performance with GPU or CPU execution
  • MDPFileManager (mdp_file_manager.py) - Handles reading and writing of molecular dynamics parameter files
  • CommandRunner (command_runner.py) - Executes GROMACS commands with real-time progress monitoring
  • GPUCommandBuilder (gpu_command_builder.py) - Constructs optimized command lines for GPU or CPU execution
  • SimulationWorker (simulation_worker.py) - Manages the complete simulation workflow in a separate thread

User Interface

  • MainWindow (gui_main.py) - The primary user interface with input controls and progress visualization
  • LogSignal - Connects background processes to the UI for real-time logging

Supporting Files

  • main.py - Application entry point
  • gambar.py - Contains encoded image resources
  • icon.ico - Application icon for Windows

Recommended Workflow

  1. System Preparation
    Generate input files (.gro, .top, .mdp) using CHARMM-GUI:
    • Recommended versions: CHARMM-GUI v3.8+
    • Use "GROMACS" format during export
    • Include all required force fields
  2. Application Setup
    # Clone repository
git clone https://github.com/Arifmaulanaazis/gmxauto.git
cd gmxauto
  3. Configuration
    • Load your CHARMM-GUI prepared folder
    • Select hardware configuration (GPU/CPU)
    • Set simulation duration (ns/ps)
  4. Simulation Execution
    • One-click workflow automation:
      1. Energy Minimization
      2. Equilibration
      3. Production MD
    • Automatic parameter optimization
    • Checkpoint handling for long runs
  5. Analysis
    • Outputs saved in structured directory:
      • Trajectory files (.xtc, .trr)
      • Log files (.log)
      • Checkpoint files (.cpt)
    • Compatible with common analysis tools (VMD, PyMOL, MDAnalysis)

Installation

Requirements

  • OS: Windows 10/11 (64-bit)
  • Python: 3.8+ (Recommend Anaconda/Miniconda)
  • GROMACS: Version 2025.1
    • Use the prebuilt Windows binary with CUDA GPU support from: GROMACS 2025.1 Prebuilt Windows CUDA
    • This prebuilt includes CUDA 12.1 support for NVIDIA GPUs and is compiled with Visual Studio 2022.
  • GPU Users:
    • NVIDIA CUDA Toolkit 12.0+
    • Driver version 535+
  • Python Dependencies:
    • PyQt6

Setup Steps

  1. Install Python dependencies: 
    pip install pyqt6
  2. Clone the repository: 
    git clone https://github.com/Arifmaulanaazis/gmxauto.git
cd gmxauto
  3. Configure GROMACS:
    • Gromacs Binary Location:
      • GPU version: ./gmx/ folder
      • CPU version: ./gmx_cpu/ folder
    • The application will automatically add GROMACS to your PATH when running
  4. (Optional) For GPU support:
    • Install CUDA Toolkit
    • Verify CUDA detection: 
      nvidia-smi
  5. Run the application: 
    python main.py

Usage

Starting the Application

Launch gmxauto by running:

python main.py

Interface Overview

  • Working Folder - Select the directory containing CHARMM-GUI output files
  • GROMACS Engine - Choose between CUDA (GPU) or CPU execution
  • Number of GPUs - Specify how many GPUs to use (disabled for CPU mode)
  • Number of CPU Cores - Set the number of CPU cores for computation
  • Simulation Duration - Define the length of the production simulation
  • Time Unit - Select between nanoseconds (ns) or picoseconds (ps)

Running a Simulation

  1. Configure all parameters as described above
  2. Click to begin the automated workflow
  3. Monitor progress through:
    • Overall progress bar - Shows completion across all steps
    • Step progress bar - Shows completion of the current step
    • Log output - Displays detailed information with color-coded status indicators:
      • ✅ SUCCESS - Successfully completed operations
      • ℹ️ INFO - General information messages
      • ⚠️ WARNING - Non-critical issues that may need attention
      • ❌ ERROR - Critical problems that prevent simulation progress
      • 💻 COMMAND - GROMACS commands being executed
  4. If needed, click to safely terminate the simulation
  5. Upon completion, all output files will be available in the working directory

Simulation Steps

gmxauto automatically executes the following steps in sequence:

  1. Preprocessing Minimization - Prepares the system for energy minimization
  2. Minimization - Removes unfavorable interactions and optimizes the structure
  3. Preprocessing Equilibration - Sets up the equilibration phase
  4. Equilibration - Gradually brings the system to target temperature and pressure
  5. Preprocessing Production - Prepares for the production simulation
  6. Production - Runs the main molecular dynamics simulation

Checkpoint Recovery

If a simulation is interrupted, gmxauto will automatically detect checkpoint files and offer to resume from the last saved point. This feature is particularly valuable for long production runs.

Troubleshooting

Common Issues:

  • GROMACS not found
    • Ensure GROMACS binaries are correctly placed in the gmx/ (for GPU) or gmx_cpu/ (for CPU) folders
    • Check that the folder structure includes bin/ and share/gromacs/ subdirectories
  • CUDA initialization error
    • Verify NVIDIA drivers are installed and up-to-date (minimum version 535+)
    • Confirm GPU is detected with nvidia-smi command
    • Ensure CUDA Toolkit 12.0+ is installed
  • Simulation crashes
    • Check .mdp files for compatibility with GROMACS 2025.1
    • Verify all required force field files are present
    • Examine log files for specific error messages
  • Performance issues
    • Adjust the number of GPUs and CPU cores for your hardware
    • For multi-GPU setups, ensure proper load balancing
    • Try increasing nstlist value for better performance

Logging:

  • Application logs: gmxauto.log in the application directory
  • Step-specific logs: progress_*.log and output_*.txt in the working directory
  • GROMACS outputs: step*.log in working directory

Performance Tips:

  • Use 1 GPU per 4 CPU cores for optimal balance
  • For multi-GPU setups, set Number of CPU Cores = Number of GPUs × 4
  • Allocate 2-4 GB RAM per core
  • Prefer NVMe storage for large trajectories
  • For very large systems, increase the nstlist parameter (automatically set to 300)
